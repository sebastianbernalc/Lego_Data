  3. Building: 5.000.000 unidades

La produccin de estas nuevas unidades empieza en 1 mes y no se tienen piezas en inventario.

Anlisis de la Estructura de la Base de Datos

Segn el esquema proporcionado, las tablas importantes para este anlisis son:

  • inventory_sets: Para obtener los inventarios relacionados con los sets.
  • sets: Para identificar los sets por nombre y tema (como "Star Wars", "Hroes", "Building").
  • themes: Para categorizar los temas de los sets.
  • inventory_parts: Para identificar las piezas necesarias para los sets especficos.
  • colors: Para obtener los colores de las piezas necesarias.
  • parts y part_categories: Para obtener informacin adicional sobre los tipos y categoras de piezas.

Determinacin de Cantidad y Proporciones de Colores para Produccin

Para alcanzar una produccin de 5,000,000 unidades para los temas "Star Wars", "Heroes" y "Building", se sigue un enfoque sistemtico que involucra un anlisis detallado de los datos disponibles. El proceso se desglosa de la siguiente manera:

Filtrado por Temtica

Comenzamos con la tabla themes para identificar los temas de inters. Los temas permiten focalizar las reas de produccin y reducen el rango de inters a conjuntos especficos. En el caso de LEGO, los temas son cruciales para determinar qu sets se fabrican y cul es el equipo responsable.

Relacin con Sets

Usamos la informacin de themes para filtrar los sets asociados a cada tema. Esto se logra mediante la clave fornea theme_id en la tabla sets, que se vincula con id en la tabla themes.

Filtrado de Inventarios

Una vez obtenidos los sets relevantes, filtramos los inventarios asociados a esos sets utilizando la clave fornea set_num en la tabla inventories, que corresponde a set_num en sets.

Obtencin de Piezas

Con los inventarios filtrados, conectamos con la tabla inventory_parts para obtener detalles sobre las piezas utilizadas en cada inventario. Esta tabla se une con colors a travs de la clave color_id en inventory_parts y id en colors.

Clculo de Proporciones de Colores

Finalmente, con la informacin de piezas y colores, calculamos la distribucin de colores, ajustando las cantidades segn la produccin requerida. Esto implica calcular la proporcin de cada color en base al total de piezas y escalar estas proporciones para cumplir con el objetivo de produccin.

Flujo de Trabajo Implementado

  • Ruta de Filtrado (amarillo): Incluye la seleccin de datos basados en los temas, sets, y inventarios especficos.
  • Ruta de Unin de Tablas (verde): Incluye la combinacin de datos de diferentes tablas para obtener la distribucin de colores.

Esta metodologa asegura una comprensin clara de los requisitos de colores para la produccin deseada, proporcionando una base slida para planificar y ejecutar la fabricacin de sets de LEGO.
